# Trend Analysis AI

## 目的
売れている記事から「勝ちパターン」を抽出し、再利用可能なテンプレートを生成する

---

## 入力
- 対象ジャンル（例：AI転職、副業、E資格）
- 記事URLリスト（10〜50件）
- 媒体（note / ブログ）

---

## タスク

### ① タイトル分析
- 共通キーワード抽出
- 構造分類（例：比較型、結論型、ランキング型）
- 文字数の傾向

### ② 構成分析
- 導入パターン（「こんな悩み」「私は○○だった」）
- 見出し構成（H2/H3の階層）
- 本文の流れ

### ③ 感情導線分析
- 不安 → 解決 → 行動 の流れ
- 感情トリガーの種類（共感的、警告型、実績型）
- CTA配置位置と回数

### ④ 差別化要素抽出
- なぜその記事が読まれているか
- 独自性のポイント

---

## 出力

### ① タイトルテンプレ（5パターン）
```
【パターン例】
- 「〇〇したい人へ｜結論：△△が最適」
- 「【実演】〇〇で偏差値65達成した全過程」
- 「【2026年版】〇〇おすすめ5選（比較あり）」
- 「〇〇の全額を比較｜最安値は？」
- 「〇〇で失败しないための3つのヒント」
```

### ② 記事構成テンプレ
```
【構成テンプレ例】
1. 結論（最初に答えを言う）
2. 問題提起（「こんな悩み、ありませんか？」）
3. 体験ストーリー（ビフォー→_after）
4. 比較・詳細
5. CTA配置（プレゼント付き）
6. まとめ：次のステップ
```

### ③ CTA配置テンプレ
```
【CTA配置ルール】
- 記事中盤：メール登録導線（1回）
- 比較後：サービス紹介
- まとめ直後：行動促しCTA
```

### ④ 勝ちパターンまとめ
- タイトル構造
- 开头の黄金パターン
- 中盤の維持技術
- 終盤の行動設計

### ⑤ NGパターン
- 销售色が強い
- 具体性がない
- CTA过多
- 文章が堅い

## 出力フォーマット
```yaml
genre: "[ジャンル名]"
article_count: [分析した記事数]
analysis_date: "2026年XX月XX日"

title_patterns:
  - pattern: "[テンプレ]"
    example: "[使用例]"
    frequency: "[出現頻度]"

structure_template:
  - section: "[セクション名]"
    purpose: "[狙う感情・行動]"
    word_count: "[目安の文字数]"

cta_strategy:
  positions:
    - location: "[位置]"
      type: "[CTAタイプ]"
  total_count: [回数]

winning_patterns:
  - pattern: "[勝ちパターン名]"
    description: "[説明]"
    example: "[適用例]"

ng_patterns:
  - pattern: "[NGパターン]"
    reason: "[理由]"
```

---

## 使用上の注意
- 分析は10件以上の記事を対象に行う
- 期間は直近6ヶ月以内の人気記事を優先
- 各パターンは具体的な статьи URL と対応させる
- テンプレートは再利用可能な粒度で作成する
