# Trend Analysis ワークフロー

## 目的
売れている記事の分析から「勝ちテンプレート」を生成し、
後の記事制作効率を最大化するためのワークフロー

---

## フロー概要

```
[1. URL収集] → [2. Trend Analysis AI投入] → [3. テンプレ生成]
     ↓                                          ↓
[6. Content AIで記事生成] ← [5. templates/保存] ← [4. レビュー]
```

---

## 手順詳細

### Step 1: URL収集

**対象記事の収集方法：**

| 媒体 | 収集方法 |
|------|----------|
| note | noteランキング、キーワード検索上位10〜30件 |
| ブログ | Google検索上位、SEO上位記事 |

**収集条件：**
- 期間：直近6ヶ月以内
- 件数：10〜50件
- 対象ジャンルと一致するもの

**URLリスト作成：**
```csv
url,article_title,platform,ranking,likes (optional)
https://note.com/...,タイトル,note,Top10,500
```

---

### Step 2: Trend Analysis AIに投入

**入力準備：**
```
対象ジャンル: [例：AIエンジニア転職]
媒体: note
URLリスト: [URLを10〜50件用意]
```

**Trend Analysis AIの指示：**
1. 各URLの記事内容を分析
2. タイトル・構成・CTA・感情導線を抽出
3. 共通パターンを整理
4. 再利用可能なテンプレートを生成

---

### Step 3: テンプレ生成

**Trend Analysis AIの出力：**
- タイトルテンプレ（5パターン）
- 記事構成テンプレ
- CTA配置テンプレ
- 勝ちパターンまとめ
- NGパターン

---

### Step 4: レビュー

**確認項目：**
- [ ] タイトルテンプレが5パターン以上ある
- [ ] 構成テンプレが明確である
- [ ] CTA配置位置が指定されている
- [ ] 勝ちパターンが3つ以上ある
- [ ] NGパターンが明文化されている

**品質チェック：**
- 既存の記事（note_01, note_02など）と照らし合わせる
- 実際に売れている要素が含まれているか確認

---

### Step 5: templates/ に保存

**保存先：**
```
templates/
├── seo_article_template.md  (既存)
├── note_article_template.md  (既存)
├── x_post_template.md       (既存)
├── trend_[ジャンル]_template.md  (新規)
└── output_schema.md         (既存)
```

**ファイル名ルール：**
```
trend_[ジャンル]_template.md
```

**例：**
- `trend_ai_career_template.md`
- `trend_e_qualification_template.md`

---

### Step 6: Content AIで記事生成

**記事制作フロー：**

```
1. topic_queue.csvからトピックを選択
2. 関連トレンドテンプレートを参照
3. Content AIが記事を生成
4. 品質チェック
5. 承認・公開
```

---

## 実行タイミング

| タイミング | 内容 |
|------------|------|
| 月1回 | 主要ジャンルのトレンド分析を更新 |
| 新規ジャンル開始時 | そのジャンルのテンプレを作成 |
| 記事の成果が悪い時 | 原因分析→テンプレ改善 |

---

## KPI measurement

**トラッキング項目：**
- 分析記事数
- 生成したテンプレ数
- テンプレート利用率
- テンプレート使用記事の平均パフォーマンス

---

## テンプレート例

### AIエンジニア転職トレンドテンプレート

**タイトルテンプレ：**
1. 「【実演】〇〇で偏差値〇〇達成した全過程」
2. 「〇〇と△△を比較｜最安値は？」
3. 「〇〇の全額を比較｜雰囲きたい人向け」
4. 「【2026年版】〇〇おすすめ3選」
5. 「〇〇で失敗しないための3つのヒント」

**構成テンプレ：**
```
1. 悩み提起（「こんな悩み、ありませんか？」）
2. 結論を先に（「私は○○で成功しました」）
3. 体験ストーリー（ビフォー→_after）
4. 方法論详细介绍
5. 比較（複数サービスを比較）
6. CTA配置（プレゼント付き）
7. まとめ：次のステップ
```

**CTA配置：**
- 中盤：メール登録導線（1回）
- 比較後：サービスCTA
- まとめ後：行動促し
