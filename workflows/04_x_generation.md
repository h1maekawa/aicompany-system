# ワークフロー04: X（旧Twitter）投稿生成

## 目的
	SEO記事の公開に合わせて、Xで投稿群を 生成し、記事へのクリック诱导を行う。

## 入力ファイル
- `workflows/content_queue.csv`（publish_urlが設定されている記事）
- `templates/x_post_template.md`

## 手順

### Step 1: 対象記事の抽出
1. content_queue.csv を開く
2. publish_url が設定されている recent なアイテムを抽出（最大3件）
3. 各アイテムのタイトル・URL・テーマを確認

### Step 2: 単発ポスト生成（記事あたり5本）
1. 以下の方向性で5本作成:
   - 質問型：読者 ownの問題提起
   - 共感型：同样的悩みを持つ人への共感
   - 結果型：实演 结果の提示
   - 方法提示型：具体的な解决方案
   - まとめ型：記事内容の振り返り
2. 各ポストの冒頭にフックを入れる
3. 末尾に [ARTICLE_URL] を配置
4. ハッシュタグ1-2個追加（最大3つまで）

### Step 3: スレッド生成（記事あたり1本）
1. 5-10ポスト構成で作成
2. 各ポストは280字以内に収める
3. 構成:
   - 1ポスト目：フック（共感できる悩み提示）
   - 2-9ポスト目：方法論・实演・结果の说明
   - 最終ポスト：CTA + [ARTICLE_URL]
4. リプ誘導：「コメントで聞きたいこと」を入れる

### Step 4: リプ誘導文生成（3本）
1. 10-20字程度の短い問いかけ
2. コメントを促す形式
3. 例：「你最 спрашивает のはどの部分？」「意见があればコメントで！」

### Step 5: 再配信号生成（2本）
1. 3日後用と7日後用の2本を作成
2. 50-80字程度
3. 过去記事を振り返るトーン
4. 例：「以前書いた記事が好评 снова見てみました@[ARTICLE_URL]」

### Step 6: approval_queue 登録
1. 生成したX投稿群を approval_queue.csv に登録
2. item_type = x_post_review
3. status = pending

## 出力
- 単発ポスト（記事あたり5本）
- スレッド（記事あたり1本）
- リプ誘導文（3本）
- 再配信号（2本）
- approval_queue.csv 更新

## フック文の失敗時修正ルール

| 问题 | 修正方法 |
|------|----------|
| クリック率が1%未満 | 冒頭を更强の共感表現に変更 |
| 反応がない | 質問形式，增加具体的な数字 |
| リプライがない | リプ誘導文を追加 |
| スレッドが伸びない | 1ポスト目のフックを更有り問題に改变 |
