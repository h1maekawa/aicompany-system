# ワークフロー05: 承認・公開

## 目的
	人間が確認・承認し、記事を安全に公開する。問題があれば差し戻しを行う。

## 入力ファイル
- `workflows/approval_queue.csv`
- `workflows/content_queue.csv`
- 承認対象の記事内容

## 手順

### Step 1: 承認待ちアイテムの確認
1. approval_queue.csv を開く
2. status = pending をすべて抽出
3. 各アイテムの checkpoints を確認

### Step 2: 人間が確認するチェック項目

#### コンテンツレビュー（content_review）
- [ ] 誇大表現（「必ず」「絶対」「最短」）が入っていないか
- [ ] 体験談の真偽が確認できるか
- [ ] 具体例・データが揃っているか
- [ ] CTAが3回以内か
- [ ] 内部リンクが5本以上か

#### 収益導線確認（monetize_check）
- [ ] CTA 配置场所が自然か
- [ ] A8リンク先が正しい案件か
- [ ] CTA文言が夸大でないか
- [ ] 比較表が公平か（片方を不当引き上げていないか）

#### note記事の追加確認
- [ ] 销售色が強すぎないか
- [ ] メール登録導線が自然か
- [ ] 比較記事誘導が自然か

#### X投稿の追加確認
- [ ] 営業的すぎる表現がないか（「必勝」「絶対儲かる」）
- [ ] ブランド布教だけの投稿になっていないか
- [ ] ハッシュタグが3つ以内か
- [ ]  ссылка 없는 ポストで終わっていないか

### Step 3: 承認・差し戻し判断

#### 承認する場合
1. approval_queue.csv の status を approved に更新
2. 関連 content_queue.csv の status を更新:
   - 記事なら: approved → scheduled（公開予約）
   - X投稿なら: published（ 즉시公開）
3. comment に承認理由を記録

#### 差し戻す場合
1. approval_queue.csv の status を revision_required に更新
2. comment に差し戻し理由を詳しく記載
3. owner に修正指示を送信
4. content_queue.csv の status を revision に更新

### Step 4: 公開処理（承認後）
1. scheduled 状態のアイテムをカレンダーに登録
2. 公開前に最終確認
3. 公開執行
4. publish_url を content_queue.csv に設定

## 差し戻し判定基準

| 問題 | 判断 | 处置 |
|------|------|------|
| 誇大表現あり | 即差し戻し | 文言修正 |
| A8リンク先錯誤 | 即差し戻し | リンク修正 |
| CTA4回以上 | 即差し戻し | CTA削除 |
| 営業色が強い | 差し戻し | 口調调整 |
| 具体性不足 | 差し戻し | 詳細追加 |

## 失敗時の分岐

| 状態 | 处置 |
|------|------|
| 差し戻し後3日以内修正なし | Secretaryへ報告 |
| 修正依頼に回应なし | タスクを却下 |
| 緊急と判断した場合 | 即座にSecretary判断を仰ぐ |
